Objetivo geral

Desenvolver um estimador exploratrio (uma aplicao simples em Python) que calcule as emisses operacionais de CO associadas ao consumo eltrico de data centers no Brasil, a partir de:

  • dados da matriz eltrica (fatores de emisso de CO por MWh);
  • dados de consumo de energia eltrica;
  • hipteses sobre a participao dos data centers no consumo total de energia.

Ao longo do notebook, vamos:

  1. Carregar e visualizar os fatores de emisso anuais (tCO/MWh).
  2. Carregar os dados de consumo de energia, agregando-os por ano.
  3. Relacionar consumo e fatores de emisso para obter emisses anuais totais.
  4. Estimar a parcela de emisses atribuda aos data centers.
  5. Projetar essas emisses para o futuro utilizando sries temporais e gerar grficos de visualizao.

O Projeto

Este notebook faz parte do projeto de APC e tem como foco estimar e projetar as emisses de CO associadas ao consumo de energia de data centers no Brasil.

A ideia construir um fluxo completo de anlise de dados:

  1. Ler e organizar fontes de dados pblicas (planilhas/CSV).
  2. Tratar e padronizar as informaes (anos, unidades, tipos numricos).
  3. Calcular indicadores de interesse (fatores de emisso, consumo anual, emisses).
  4. Aplicar um modelo simples de previso para explorar cenrios futuros.

O resultado final um estimador que ajuda a entender como o crescimento do consumo de data centers pode impactar o perfil de emisses do pas ao longo do tempo.

Agora vamos relacionar os fatores de emisso de CO com o consumo anual de energia.

A ideia :

  • unir, em uma nica tabela,
    • o fator de emisso anual (tCO/MWh);
    • o consumo anual de energia (MWh);
  • a partir disso, calcular a emisso total de CO por ano, usando a frmula:

$\text{emisso anual (tCO)} = \text{consumo anual (MWh)} \times \text{fator de emisso (tCO/MWh)}$

Na clula de cdigo seguinte, fazemos essa juno por ano e criamos uma coluna com o valor estimado de emisses totais do sistema eltrico.

In[18]:
# Carregar os arquivos
consumo = pd.read_csv("input/consumo_anual_MWh.csv")
fatores = pd.read_csv("input/fatores_emissao.csv")

# Garantir que a coluna 'ano' seja numrica
consumo["ano"] = consumo["ano"].astype(int)
fatores["ano"] = fatores["ano"].astype(int)

# Fazer o merge com base no ano
df_final = consumo.merge(fatores, on="ano", how="inner")

# Calcular a emisso anual em tCO2
df_final["emissao_tCO2"] = (
    df_final["consumo_anual_MWh"] * df_final["fator_emissao_tCO2_MWh"]
)

# Salvar o resultado final
df_final.to_csv("input/emissoes_anuais_2006_2024.csv", index=False)

Tendo a emisso anual total do sistema eltrico, podemos prosseguir para as estimativas relacionadas ao consumo de data centers.

Para isso, usamos como referncia o estudo da Brasscom sobre o consumo de energia em data centers, que aponta que:

  • Em 2024, os data centers representaram cerca de 1,7% do consumo total de energia eltrica no Brasil.
  • A projeo para 2029 que essa participao chegue a aproximadamente 3,6%.

No cdigo a seguir, fazemos basicamente trs coisas:

  1. Interpolamos a participao dos data centers ao longo dos anos (por exemplo, ligando 1,7% em 2024 a 3,6% em 2029 e estendendo essa lgica at 2030).
  2. Aplicamos essa participao sobre a srie de emisses totais, obtendo a emisso anual atribuda apenas aos data centers.
  3. Preparamos essa srie anual de emisses de data centers para ser usada em um modelo de previso de sries temporais (Prophet).

Com isso, passamos a ter uma srie histrica de emisses especficas de data centers, pronta para ser projetada para anos futuros.

# Construir uma srie nica: real + prevista para incluir os data centers

# Srie vinda do Prophet
previsao = forecast[["ds", "yhat"]].copy()
previsao["ano"] = previsao["ds"].dt.year

# intervalo de interesse
previsao = previsao[previsao["ano"].between(2006, 2030)].copy()

# Pegar ano + emisso real da base calculada
hist = df_final[["ano", "emissao_tCO2"]].copy()
hist = hist[hist["ano"].between(2006, 2024)].copy()

# Juntar real com previsto
previsao = previsao.merge(hist, on="ano", how="left")

# emissao_tCO2_real (quando existe) OU yhat (quando no tem real)
previsao["emissao_total_tCO2"] = previsao["emissao_tCO2"].fillna(previsao["yhat"])

# CURVA DE PARTICIPAO DOS DATA CENTERS (em frao, no %)
curva_dc = pd.DataFrame({
    "ano": [
        2006, 2007, 2008, 2009,
        2010, 2011, 2012, 2013,
        2014, 2015, 2016, 2017,
        2018, 2019, 2020, 2021,
        2022, 2023, 2024, 2025,
        2026, 2027, 2028, 2029
    ],
    "participacao_DC": [
        0.0035, 0.0038, 0.0041, 0.0045,   # 0,35%  0,38%  0,41%  0,45%
        0.0055, 0.0065, 0.0075, 0.0090,   # 0,55%  0,65%  0,75%  0,90%
        0.0100, 0.0110, 0.0120, 0.0130,   # 1,00%  1,10%  1,20%  1,30%
        0.0140, 0.0150, 0.0155, 0.0160,   # 1,40%  1,50%  1,55%  1,60%
        0.0163, 0.0165, 0.0170, 0.0200,   # 1,63%  1,65%  1,70%  2,00%
        0.0240, 0.0280, 0.0320, 0.0360    # 2,40%  2,80%  3,20%  3,60%
    ]
})

# Juntar a curva com a previso (left join por ano)
previsao = previsao.merge(curva_dc, on="ano", how="left")

# Para anos sem valor explcito na curva
# manter o ltimo valor conhecido (3,6%)
previsao["participacao_DC"] = previsao["participacao_DC"].ffill()

# Emisso dos data centers = emisso total * participao
previsao["emissao_DC_tCO2"] = (
    previsao["emissao_total_tCO2"] * previsao["participacao_DC"]
)
